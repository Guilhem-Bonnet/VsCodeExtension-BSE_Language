Démarrage

  • Autoriser la création automatique d'invités

    • Le blocage est l'option par défaut. On entre un nom d'utilisateur inconnu, il a été ajouté dans les paramètres généraux.

  • Vérifier les soldes comptables au 1er démarrage

    • Ouverture d'une application pour la 1ère fois de la journée, si dans les Paramètres avancés de l'application il est coché "Vérifier les soldes comptables au 1er démarrage", la compta est vérifiée.

  • Mise à jour des volumes au 1er démarrage

    • Si dans les Paramètres avancés de l'application il est coché "Mise à jour des volumes au 1er démarrage", les Volumes seront vérifiés

Adresses

  • Saisie des adresses selon la norme INSEE

    • Avant toute exécution, il faut s’assurer que la base BELAIR dispose de:

    • la table de code INSEECODEB
    • la table de code INSEECODEV
    • Deux script BSE situés dans le dossier : C:\Belair4\APPXXX\BASScripts\Std
      • Normalisation adresses.bs
      • Remplacement b_adrN par b_adresseN dans les modeles de lettre.bs

    FAIRE UNE SAUVEGARDE BELAIR

    Execution

    • Ouvrir BSEDEV.exe
    • Régler les paramètres : menu Run / Paramètres : cocher "Log on to Belair" et indiquer User : SUPERVISEUR ainsi que son mdp dans Password
    • Ouvrir le fichier C:\Belair4\APPXXX\BASScripts\Std\Normalisation adresses.bs puis executer avec F9
    • Un fichier .csv va lister les adresses migrées et celles à corriger
    • Ouvrir le fichier C:\Belair4\APPXXX\BASScripts\Std\Remplacement_b_adrN_par_b_adresseN_dans_les_modeles_de_lettre.bs puis executer avec F9
  • Gestion des codes pays selon la norme ISO 3166

    • Les codes de pays peuvent être représentés par un code à deux lettres (alpha-2) recommandé pour l'usage général, un code à trois lettres (alpha-3) associé plus étroitement au nom de pays, et un code à trois chiffres (numeric-3), utile lorsque les codes doivent être compris dans des pays n'utilisant pas l'alphabet latin.

  • Alimentation automatique du fichier des communes

    • Le fait de saisir un code postal et une commune en portefeuille alimente automatiquement le fichier des communes. Option accessible dans les paramètres généraux pour avoir le choix au sujet de cette mise à jour. (Positionnée a "Vrai" au début pour garder la gestion existante)

Sinistres

  • Blocage sur date de survenance sinistre
    • Vérifier si la date de survenance est supérieure à la date d'ouverture
    • Vérifier que le contrat est en cours
    • Vérifier que le risque est couvert (date e/s du risque).
    • Si le contrôle n'est pas coché, il y aura un message d'alerte, et une validation à faire pour pouvoir continuer.
    • Si le contrôle est coché, il ne sera pas possible de créer un sinistre dont la date de survenance correspond aux cas cités.
  • Vérification du solde des comptes sur la clôture d'un sinistre

    • Définition

  • Contrôle sur les garanties sinistrées, leurs règlements et leurs évaluations : Cette option va déclencher la cohérence entre les règlements et les évaluations.
    • Lors de la saisie des évaluations, si les évaluations sont inférieures aux règlements, alors il n'est pas possible de les enregistrer une erreur s'affiche: 
      Evaluation inferieure au règlement ne peut pas être enregistré
    • Lors de la clôture d'un sinistre, si les évaluations ne correspondent pas aux règlements, alors une alerte s'affiche:

      Attention, avant clôture du sinistre les évaluations des garanties sinistrées ne correspondent pas aux totaux des règlements effectuées.
      Voulez-vous que le programme corrige automatiquement toutes les évaluations ?'

    • Lors de la saisie des garanties sinistrées, si une garantie n'a pas d'évaluation, un message s'affiche:

      Attention, il est obligatoire de définir l'évaluation compagnie pour chaque garantie sinistrée

    • Lors de la saisie d'un règlement, si les évaluations ne sont pas suffisantes, un message s'affiche:

      Evaluation insuffisante, le règlement ne peut pas être enregistré

  • Limiter au SUPERVISEUR l'effacement des sinistres dans la période de conservation

    • Permet au SUPERVISEUR uniquement l'effacement des sinistres dans la période de conservation

Calcul

  • Vérifications de champs vides :

    • Une nouvelle option est prévue pour pouvoir vérifier si un champ est vide : Outils | Paramètres | Paramètres généraux - onglet Avancés : option "Vérifications de champs vides".

  • Arrondi historique :
    • Option activée : on ajoute toujours la valeur de 0.000001, ce qui rend tout arrondi à plus de 5 décimales faux.
    • Option désactivée : cette valeur de 0.000001 n'est plus ajoutée, les résultats sont justes.
  • Transformation en Zéro des formules de calcul numérique rendant un résultat vide:

    • Permet de contrôler le comportement des formules de calcul.

    • Option activée : les résultats intermédiaires sans valeur (Null) sont transformés en ZERO
    • Option désactivée : si un résultat intermédiaire est Null, la formule finale rend Null
