Anonymisation de la base de données


Afin d'être conforme au RGPD, vous pouvez être amené à fournir une copie de votre base de données à un tiers, ne contenant aucune donnée personnelle sensible (nom/prénom, n. de téléphone, références bancaires, etc..

Belair intègre deux fonctions de maintenance permettant de procéder à l'anonymisation complète d'une base.
Ces fonctions sont accessibles uniquement aux adminstrateurs.

RGPDFIELDS

Cette fonction analyse le modèle de données de votre base, y compris vos données spécifiques (exrtensions, ...) et recense tous les champs sensibles (n. de téléphone, email, etc...) en plus des champs déclarés sensibles par défaut (nom, prénom, Iban, etc...).
La liste des champs est stockée dans la table de codes RGPDFIELDS

En cas de besoin, vous pouvez ajouter dans cette table tout champ jugé sensible, non repéré du fait de sa nature, comme par exemple un champ alphabétique simple dans une extension peronnelle, utilisé pour stocker un nom, une adresse email, etc...

RGPDANONYME

Lancement de la fonction d'anonymisation de la base.

Cette opération étant irréversible, une confirmation sur le lancement de l'opération est demandée.
Vous demandez l'anonymisation de la base.
Ceci signifie que toutes les données personnelles vont être effacées, de manière DEFINITIVE
