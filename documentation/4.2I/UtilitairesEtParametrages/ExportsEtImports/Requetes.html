Les requêtes

Les opérations suivantes s'effectuent à partir de l'éditeur de requêtes.

Exportation :

A partir de l'application d'origine, il existe trois façons de procéder :

  • Il est possible d'exporter toutes les requêtes en demandant :


  • Cliquer sur pour afficher les classeurs.
    Faire un clic-droit sur le classeur à exporter et demander "Exporter Classeur".

  • Cliquer sur pour afficher les classeurs.
    Ouvrir le classeur contenant la (ou les) requêtes(s) devant être exportée(s).
    Ouvrir la requête à exporter. Les requêtes ne peuvent s'exporter qu'une par une.
    Demander :


Le nom du fichier d'export est TRFREQ, cliquer dans (les fichiers d'export sont stockés implicitement dans le répertoire "travail" du poste mais il est possible d'indiquer un autre chemin de réception).
Lorsque le fichier est constitué, un message apparaît : Export terminé. Cliquer sur

Plusieurs fichiers sont alors créés : TRFREQ.db, TRFREQ.MB et PARAMx1x.DB.


Importation :

A partir de l'application d'arrivée, demander :

.

Le fichier TRFREQ est proposé dans le répertoire "travail" du poste, cliquer dans (si les fichiers ont été stockés à un autre endroit, il faut ouvrir le répertoire les contenant avant de cliquer sur ).

Le contenu du fichier apparaît dans une fenêtre vous permettant de sélectionner la (ou les) requête(s) à importer. Dans ce cas, un bouton est disponible afin de sélectionner le classeur de réception des requêtes sélectionnées.
Cocher "Avec" ou "Sans" remplacement des préexistantes.
Lorsque cette sélection est effectuée, cliquer sur un des boutons ou selon d'option choisie.

Lorsque l'importation est terminée, un message apparaît Import terminé. Cliquer sur .
