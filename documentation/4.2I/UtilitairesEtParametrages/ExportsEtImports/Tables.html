Les tables

Les opérations suivantes s'effectuent dans Outils | Paramètres | Modélisation de l'application.

Exportation

A partir de l'application d'origine, demander Fichier | Gestion des tables.
Sélectionner la table à exporter et demander Tables | Exporter la structure.
Le nom du fichier d'export est XXXXYYY.BAI (où XXXX est le nom de la table et YYY le suffixe de votre application), cliquer dans Enregistrer (les fichiers d'export sont stockés implicitement dans le répertoire "travail" du poste mais il est possible d'indiquer un autre chemin de réception).

Une question est alors posée lorsque la table est rattachée à un dessin d'écran : "Vous venez d'exporter une table d'extension. Voulez-vous exporter ses éventuelles images écran ?"

  • A la réponse NON, la fonction s'arrête là.
  • A la réponse OUI, s'enchaîne l'exportation de l'écran constituant un fichier d'export nommé TRFECRAN, cliquer sur Ouvrir (les fichiers d'export sont stockés implicitement dans le répertoire "travail" du poste mais il est possible d'indiquer un autre chemin de réception).

Plusieurs fichiers sont alors créés : XXXXYYY.BAI (pour la table) et TRFECRAN.MB, TRFECRAN.db (pour l'écran rattaché).

Importation

A partir de l'application d'arrivée, demander Fichier | Gestion des tables.
Eventuellement, demander ensuite Tables | Importer une structure.
Sélectionner le fichier XXXXYYYY.BAI dans le répertoire "travail" du poste, cliquer dans Ouvrir (si les fichiers ont été stockés à un autre endroit, il faut ouvrir le répertoire les contenant avant de cliquer sur Ouvrir).

  • "Importer sur quel code" : taper XXXX (où XXXX est le nom de la table). Cliquer sur OK.
    Si la table existe déjà dans l'application d'arrivée, un message s'affiche immédiatement : "Le code XXXX est déjà associé à la table XXXXYYY - Voulez-vous continuer et modifier ainsi la structure de cette table"
    • A la réponse NON, la fenêtre du choix du nom de la table est reproposée afin de taper un nouveau nom pour la table et l'importer.
    • A la réponse OUI, une nouvelle question est posée si la table importée est reliée à une image écran :
      "Vous venez d'importer une table d'extension. Voulez-vous importer ses éventuelles images écran."
      • A la réponse NON, la fonction s'arrête là et les images écran ne sont pas importés.
      • A la réponse OUI, le fichier TRFECRAN est proposé dans le répertoire "travail" du poste, cliquer dans Ouvrir (si les fichiers ont été stockés à un autre endroit, il faut ouvrir le répertoire les contenant avant de cliquer sur Ouvrir).
        Le contenu du fichier apparaît dans une fenêtre vous permettant de sélectionner l' (ou les) écran(s) à importer. Cocher "Avec" ou "Sans" remplacement des préexistantes".
        Lorsque cette sélection est effectuée, cliquer sur un des boutons Importer les sélectionnées ou Tout Importer selon d'option choisie.

Lorsque l'importation est terminée, un message apparaît "Import terminé". Cliquer sur OK.
