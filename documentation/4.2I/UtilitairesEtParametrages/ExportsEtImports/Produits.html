Les produits

L'exportation et l'importation d'un produit permettent de passer d'une application BELAIR à une autre, le descriptif du produit, ses garanties, ses couvertures, les indices utilisés par le produit, le détail de la tarification, les formules utilisées par la tarification (même celles appelées indirectement).

Ne sont par exportés : les tables de codes utilisées dans les formules, les taux de TVA, les champs, les tables de valeurs des champs.
Concernant les tables de codes, on trouve dans l'état récapitulatif lancé à la fin de l'export, la liste des tables utilisées dans chacune des formules exportées. Il convient alors d'effectuer un export de tables de codes.

Les opérations suivantes s'effectuent à partir de :


Exportation

Onglet :

Sélectionner le produit à exporter.



Le nom du fichier est donné : PROD_xxxxx.XML

  • PROD = "produit"
  • xxxxx = le nom du produit
  • XML = suffixe obligatoire pour l'exportation

Demander ensuite

La fenêtre "Export d'un produit vers un fichier" s'ouvre :
  • "Nom du produit" : c'est le nom du produit sélectionné qui s'affiche implicitement mais il est possible à ce moment d'en choisir un autre. Quoiqu'il en soit l'exportation n'est possible que produit par produit.
  • "Nom du fichier" : le chemin peut être modifié. Le nom du produit également MAIS le suffixe en XML doit être conservé.
Valider par

Un rapport d'exportation s'affiche avec possibilité de l'imprimer.

Un fichier est alors créé : PROD_xxxxx.XML

Importation



Aller chercher le fichier à importer (en suffixe .XML) dans le répertoire où il est stocké OU taper le "Nom" du fichier et

La fenêtre "Import d'un produit" s'ouvre :

  • "Nom du fichier" : le nom du fichier à importer s'affiche tel qu'on a été le rechercher plus haut
Valider par

La fenêtre "Paramétrage des compagnies" s'ouvre :
Le numéro de la compagnie principale du produit à importer s'affiche et il appartient à l'utilisateur d'indiquer le code de la compagnie principale que devra prendre le produit dans l'application d'arrivée (si le code est différent).

Un message de confirmation s'affiche si on constate des doublons dans les tables avant d'effectuer l'importation. Si on confirme l'importation, les valeurs en double seront écrasées par l'import, sauf pour les compagnies qui elles, seront conservées.
