SQL:NEW(Query)

Création d'une requête

Le texte SQL associé doit être inséré immédiatement en dessous, chaque ligne de commandes devant commencer par *.
Query est le nom de la requête qui sera utilisé pour les autres commandes

SQL:NEW(MONSQL)
*SELECT COUNT (*) FROM [TIERS]

SQL:OPEN(Query)

Ouverture d'une requête donnant un résultat

SQL:OPEN(MONSQL)
Le nombre d'enregistrements retournés est mis dans : RESULT

SQL:NEXT(Query)

Passage à l'enregistrement suivant

SQL:NEXT(MONSQL)
Si fin de fichier RESULT = 0, sinon RESULT = 1

SQL:CLOSE(Query)

Fermeture de la requête

SQL:CLOSE(MONSQL)

SQL:FREE(Query)

Suppression de la requête (important : un FREE pour chaque NEW)

SQL:FREE(MONSQL)

SQL:EXEC(Query)

Execution d'une requête de type 'UPDATE'

SQL:EXEC(MONSQL)

SQL:NEWEXT(Query)

Permet d'accéder à une base autre que la base Belair

Dans l'automate de traitement de données, dès lors que la source de données est un SGBD autre que la base BELAIR (connexion à un alias),
il faut pouvoir créer un SQL connecté à cette base et non pas à APPxxx

SQL:NEWEXT(Alias, MONSQL)

Voici un Exemple:

Déclaration de l'Alias à partir de l'onglet "Description",
Cocher SGBD
Dans le menu déroulant Alias, sélectionner [Ajouter] puis cliquer sur [Editer] pour renseigner les paramètres définissant la base sur laquelle travailler.

il est important d'utiliser SQL:FREE pour libérer la connexion

Attention les Connections sont identifiées par le nom de l'alias il est donc possible de se connecter plusieurs fois à la même Base si l'alias est différent.

Alias1 
server name=bai-dev-dim
;Base :
database=d:\belair42\appdei\appdei.idb
;Identifiants :
user name=SYSDBA
password=masterkey
Alias2 
server name=bai-dev-dim
;Base :
database=d:\belair42\appdei\appdei.idb
;Identifiants :
user name=SYSDBA
password=masterkey

On a 2 connections différentes à la même DataBase il n'y a aucun lien entre les deux connections.
On peut être connecter à une infinité de base en même temps.
