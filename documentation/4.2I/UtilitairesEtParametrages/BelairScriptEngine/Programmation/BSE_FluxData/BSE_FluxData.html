BSE Flux Data


Pour normaliser la gestion des flux de données de type Data (flux transmis en paramètres d'un script), il faut respecter les règles ci-dessous :

  • Si la balise est absente, alors :
    Champ non mis à jour, ou non renseigné.
  • Si <balise></balise> ou <balise/>, alors :
    Champs mis à jour avec comme valeur " ".
  • Si <balise>Test</balise>, alors :
    Champs mis à jour avec le texte "Test".
  • Si <balise IsNull=True/></balise>, alors :
    Champs mis à jour avec la valeur "Null".

Exemple d'un flux Data

<data>
  <login>(obligatoire) : Login utilisateur</login>
  <passwordhash>(obligatoire) : password</passwordhash>
  <credentialtype>(obligatoire) : type Credential </credentialtype>
  <fullname>(optionnel) : Nom Complet </fullname> //peut être Null
  <style>(optionnel) : Style </style> //peut être Null
  <palette>(optionnel) : Palette </palette> //peut être Null
  <language>(obligatoire) : language  ex "fr" </language>
  <workphone>(optionnel) : Work Phone </workphone> //peut être Null
  <cellphone>(optionnel) : Cell Phone </cellphone> //peut être Null
  <disabled>(obligatoire) : disabled </disabled>
  <email>(optionnel) : Mail default </email>
  <profilcrm>(optionnel) : Workgroup </profilcrm>
</data>
