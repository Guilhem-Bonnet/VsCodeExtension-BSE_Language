Library FileCsvLib 

function File_ConvertCsvToXml(inputfilename, encoding, ReadColumsNames, SkipFirstLine, Columns, RowName, outputfileName)

Retourne un flux XML (ou un fichier XML) du contenu d'un fichier CSV


function File_ConvertCsvToArray(inputfilename, encoding, ReadColumsNames, SkipFirstLine, Columns)

Retourne un tableau représentant le contenu d'un fichier CSV


Paramètres communs aux 2 fonctions :
  • inputfilename (string) : obligatoire : fichier CSV en lecture
  • encoding (string) : obligatoire : encodage du fichier, parmi les valeurs suivantes
    • ANSI
    • UTF8 ou UTF-8
    • UTF16 ou UTF-16
  • ReadColumnNames (booléen) : obligatoire. Si True, les noms des colonnes sont automatiquement attribués selon la 1ere ligne du fichier (en-tête)
  • SkipFirstLine(booléen) : obligatoire. Si True, on ignore la 1ere ligne (en-tête). Forcé à True si ReadColumnNames = True
  • Columns (string) : optionnel. si renseigné, liste des noms de colonnes à créer, séparés par des ";"

    • Ce parametre permet d'ignorer certaines colonnes du CSV et de forcer le type de chaque colonne (par défaut, type=String)
    • Exemple 1 : columns := "numero;nom;;prenom;date_naissance" => Seules les colonnes 1,2,4 et 5 du fichier sont lues
    • Exemple 2 : columns := "numero=L;nom;;prenom;date_naissance=D;salaire=E"
      • => Seules les colonnes 1,2,4,5 et 6 du fichier sont lues
      • numero est de type Integer, date_naissance est de type TDateTime et salaire est de type Float (même codification que dans Baiimp : A,L,D et N)

Paramètres spécifiques à File_ConvertCsvToXml

  • RowName (string) : optionnel, donne le nom de la balise du résultat ("row" par défaut)
  • outputfileName (string) : optionnel, si non vide, le résultat est enregistré dans le fichier et la fonction retourne le nom du fichier créé à la place du flux XML
