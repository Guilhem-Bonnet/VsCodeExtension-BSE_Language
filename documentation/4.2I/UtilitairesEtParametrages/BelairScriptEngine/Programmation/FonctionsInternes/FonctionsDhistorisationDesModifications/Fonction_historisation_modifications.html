Fonctions d'historisation des modifications

Histo:Enable

La fonction active l’historisation des modifications pour la connexion en cours.


Remarques :

  • Si aucun trigger n'a été déclenché la fonction n'aura aucun effet.
  • Par défaut les triggers sont toujours actif donc Histo:Enable va de pair avec la fonction Histo:Disable


Paramètres d'entrée :
Aucun.


Résultat :
Aucun.


Exemples : 

Local tiersHisto, tiersNoHisto
Histo:Disable()
tiersHisto := Obj:GetByKey("TIERS", 1010)
tiersHisto.RSociale := "Pas Historisé" 
Obj:Save(tiersHisto)
Histo:Enable()

tiersNoHisto := Obj:GetByKey("TIERS", 1005)
tiersNoHisto.RSociale := "Historisé" 
Obj:Save(tiersNoHisto)                                                    

return 0


Résultat :

Ici les modifications du tiers 1005 seront tracé mais les modifications du tiers 1010 ne le seront pas..


Histo:Disable

La fonction désactive l’historisation des modifications pour la connexion en cour.


Remarques :

  • Si aucun trigger n'a été déclenché la fonction n'aura aucun effet.
  • A la fin de l’exécution d’un script les triggers seront automatiquement réactivés.


Paramètres d'entrée :
Aucun.


Résultat :
Aucun.


Exemples : 

Local tiersHisto, tiersNoHisto

Histo:Disable()
tiersHisto := Obj:GetByKey("TIERS", 1010)
tiersHisto.RSociale := "Pas Historisé" 
Obj:Save(tiersHisto)
Histo:Enable()

tiersNoHisto := Obj:GetByKey("TIERS", 1005)
tiersNoHisto.RSociale := "Historisé" 
Obj:Save(tiersNoHisto)                                                    

return 0


Résultat :

Ici les modifications du tiers 1005 seront tracé mais les modifications du tiers 1010 ne le seront pas.
