Langage de scripts


Types de fichiers


Les fichiers ont pour extension : ".bs", par défaut, peu importe leur type.

On reconnaît deux types de fichiers :

  • Un script : un fichier de script contenant les programmes.
  • Une bibliothèque : un fichier des sous-programmes/fonctions qui peuvent être utilisés dans les autres scripts.

Structure du script


Chaque script peut avoir des sections "[nom de section]" destinées à différents buts.
Un script sans aucune section est traité comme un programme/bibliothèque seulement.


Section Designation
Script Le programme même
Description La description du script

Structure du programme


Le programme contient la partie principale du code (homologue de la fonction main() en C/C++), suivie éventuellement par les sous-programmes.

Au contraire, une bibliothèque ne doit contenir que des sous-programmes.



Print("Hello, World!")

ou en utilisant un sous-programme :

Hello()

FUNCTION Hello
  Print("Hello, World!")
RETURN

Exécutez ce script dans l'environnement BSEDev ou depuis la ligne de commande avec BSEExec :

BSEExec.exe /D /:I:BSEExec_Hello.bs

Bibliothèques


Structure


Une bibliothèque doit commencer par la déclaration "LIBRARY", suivi du nom correspondant à celui du fichier script.
Cette déclaration suivie par les sous-routines.


LIBRARY MyLib 
// Le fichier est 'MyLib.bs' !

FUNCTION Hello
  Print("Hello, World!")
RETURN


Utiliser dans les scripts


Afin d'utiliser les fonctions d'une bibliothèque, le programme doit demander de les utiliser avec l'instruction "USES".
"USES" doit toujours être en tête du programme.

USES MyLib 

Hello()

Il est possible d'utiliser plusieurs bibliothèques en même temps.

USES TiersLib, ContratLib 
...
ou 
USES TiersLib
USES ContratLib 
...
