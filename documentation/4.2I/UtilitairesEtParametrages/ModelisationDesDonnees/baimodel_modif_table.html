Modification d'une table

Par défaut, les éléments sont en lecture seule.
Positionnez-vous sur la table concernée, faites Clic-Droit et sélectionnez la fonction 'Débloquer' pour passer en mode édition sur cette table.

Pour créer un nouveau champ:

Faites Clic-Droit sur la ligne 'Champs' et sélectionnez la fonction 'Nouveau' ou tapez [Ctrl]+N
Remplissez les propriétés du nouveau champ

Pour modifier les propriétés d'un champ:

Positionnez-vous sur la ligne du champ concerné et modifier ses propriétés

Certaines modifications sont impossibles et seront refusées lors de la validation du modèle.

Pour copier toutes les propriétés d'un champ sur un nouveau champ:

Positionnez-vous sur la ligne du champ concerné et sélectionnez la fonction 'Cloner'

Pour supprimer un champ:

Faites Clic-Droit sur la ligne du champ concerné et sélectionnez la fonction 'Supprimez' ou tapez [Ctrl]+Suppr
Remplissez les propriétés du nouveau champ

RAPPEL.
Les champs dont le nom est B_xxxxx font partie du modèle standard de BELAIR.
Ne tentez jamais de modifier un champ du modèle standard de BELAIR.
De même, lorsque vous ajoutez des champs à une table existante, ou dans uen table que vous créez, ne créez pas de champ avec un nom du type B_xxxxx

Pour créer un nouvel index secondaire:

Faites Clic-Droit sur la ligne 'Index' et sélectionnez la fonction 'Nouveau' ou tapez [Ctrl]+N
Remplissez les propriétés du nouvel index secondaire.

(cette fonction n'est utile que pour créer des index composés de plusieurs champs; sinon il suffit de modifier la propriété 'indexé' du champ concerné)

Pour modifier les propriétés d'un index secondaire:

Positionnez-vous sur la ligne de l'index concerné et modifiez ses propriétés.

Pour copier toutes les propriétés d'un index sur un nouvel index:

Positionnez-vous sur la ligne de l'index concerné et sélectionnez la fonction 'Cloner'

Pour copier toutes les propriétés d'une table sur une nouvelle

Positionnez-vous sur la ligne de la table concernée et sélectionnez la fonction 'Cloner' Une nouvelle table TAB_xxx

Le clonage d'une table est à manipuler avec précaution.
Si par exemple, vous clonez une extension sur une nouvelle, vous devrez respecter les normes de nommage et de relations de cette nouvelle table, en fonction de sa destination.
