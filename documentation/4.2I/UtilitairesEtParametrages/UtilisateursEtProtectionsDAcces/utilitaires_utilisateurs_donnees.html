Protection des données

La protection des données permet de protéger certains champs pour des groupes de limitation. Les données peuvent être protégées en "lecture seule" (aucune modification possible par les utilisateurs faisant partie d'un groupe) ou en "invisible" (la valeur du champ ne s'affiche pas pour les utilisateurs faisant partie d'un groupe).

Mise en œuvre : Outils | Paramètres | Modélisation de l'application

Protections des données

Les tables de BELAIR s'affichent dans la colonne de gauche. Pour ouvrir une table cliquer dans le pro33 de celle-ci et les champs s'affichent. Sélectionner ensuite le champ à protéger.

Trois colonnes sont affichées sur la droite : "Non protégé", "Lecture seule" et "Invisible"

Les groupes de limitation s'affichent dans la colonne "Non protégé". Pour définir que le champ sélectionné doit être protégé, glisser le groupe dans la colonne correspondante.

Rendre invisible le montant des commissions (sur la Prime de base et la prime quittancée) :

Menu Modélisation | Fichier | Protection des données
