Protection des types d'encaissement / de reversements

Le SUPERVISEUR peut limiter l'utilisation de certains types d'encaissement à certains groupes de limitation.
En effet, dans la fonction Gestion | Quittancement | Paramètres généraux

Onglets Comptabilité | Encaissements, Retours à compagnie, Reversements/Rétrocessions, un bouton permet de protéger certains types.

Conclusion : si un utilisateur du groupe en question veut faire des encaissements/reversements, seuls les types lui étant autorisés s'afficheront.

Limitation d'accès pour les utilisateurs au type d'encaissement "Effacement de quittance" :

Le SUPERVISEUR peut maintenant limiter l'utilisation de certains types d'encaissement à certains groupes de limitation.
En effet, dans la fonction Gestion | Quittancement | Paramètres généraux – Onglets Comptabilité | Encaissements il convient de créer un type d'encaissement ayant pour code @ (arobase) puis de cliquer sur le cadenas afin d'indiquer les utilisateurs n'ayant pas accès à ce type d'encaissement. De ce fait, l'utilisateur appartenant à ce groupe, n'aura plus la possibilité d'effacer une quittance en portefeuille.

Limitation d'accès pour les utilisateurs à la remise en impayé d'une quittance :

Il est possible d'interdire à un groupe d'utilisateur la possibilité de remettre une quittance en impayé. Pour cela, il suffit de créer un type d'encaissement ayant pour code * (astérisque) et pour libellé 'Remise en impayé'. Il suffit ensuite de lui appliquer la limitation d'accès comme à n'importe quel autre type d'encaissement.
