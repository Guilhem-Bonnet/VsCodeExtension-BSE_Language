Conditions

SI valeur1 { ET / OU valeur2 { ET / OU valeur3 ….. } } incomplet

Valeur1, valeur2, valeur3 : tests rendant VRAI ou FAUX (valeurs booléennes)

Exemples de valeur booléenne :

?DPP("b_age") VRAI si age ?CONT("b_tacite") L'information "sans tacite reconduction" est de type booléen, rendant VRAI ou FAUX

Les conditions permettent de piloter le déroulement d'une formule

Exemples d'utilisation dans une formule alphabétique" :

SI ?DPP("b_sexe") = "M" ;>HOMME // débranchement selon le sexe
SI ?DPP("b_sexe") = "F" ;>FEMME // débranchement selon le sexe
="sexe inconnu" // résultat pour les personnes mal saisies

:HOMME // la formule reprend ici pour les hommes
SI ?DPP("b_age") > 60 ;>TROPVIEUX // limite d'age = 60 pour les hommes
SI ?DPP("b_age") > 25 ;="OK" ;= "trop jeune" // "OK" si age > 25 ans, "trop jeune" sinon

:FEMME // la formule reprend ici pour les femmes
SI ?DPP("b_age") > 55 ;>TROPVIEUX // limite d'age = 55 pour les femmes
SI ?DPP("b_age") > 30 ;="OK" ;= "trop jeune" // "OK" si age > 30 ans, "trop jeune" sinon

:TROPVIEUX ="personne trop âgée"
