Actualisation / Enregistrement des données d'une table

REFRESH("code")

Permet d'actualiser les données d'une table, dans une formule de calcul.
Cette fonction est particulièrement utile lorsque des modifications de données sont faites par SQL (sinon, la table ne voit pas les modifications en temps réel)

Paramètre :

"code" : nom de la table à mettre à jour.

Exemple d'utilisation :

La table XDGRI est une extension multiple et un bouton relié à une formule de calcul en SQL permet d'initialiser cette table.
L'instruction REFRESH à la fin, permet de voir la grille se remplir (sinon, il fallait quitter l'écran et revenir).

sql:new(lprod)
*select b_codeprod from [prod]
...............
..............
:FIN
SQL:free(lprod)
REFRESH("XDGRI")
=IY

WINMES(VERIFONGLETS)

Permet d'actualiser l'affichage d'onglets conditionnés par une formule de calcul.
Cette commande est utilisable dans une condition de saisie d'un champ ou dans une formule de calcul déclenchée par un bouton.
(Peut être utilisé après la fonction REFRESH)

OK

Enregistrement (commit) des tables dont un champ au moins a été modifié par la formule de calcul

SaveAll

Enregistrement (commit) de toutes les tables en cours d'édition, même si elles n'ont pas été modifiées par la formule de calcul.
Cette fonction rend VRAI ou FAUX

Exemple d'utilisation :
IAX := SAVEALL
