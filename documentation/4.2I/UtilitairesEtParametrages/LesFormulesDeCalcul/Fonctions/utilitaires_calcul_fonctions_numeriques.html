LES ARRONDISincomplet

Règle pour la fonction de calcul ARRONDI(montant, type, niveau)

    si niveau arrondi  6 , on applique la "nouvelle règle"

Ainsi, on ne touche à rien dans le résultat des fonctions programmées (les arrondis sont souvent sur des valeurs monétaires qui n'ont jamais d'arrondi > 4 décimales) mais on règle le problème pour les autres cas éventuels

Cela permet de bien séparer les deux méthodes de calculs d'arrondis, et donc des soucis dans le futur pour chercher à comprendre pourquoi l'arrondi est parfois juste, ou parfois faux, selon le nombre de décimales demandées.

La modification du programme consiste à ajouter une option Arrondi historique, disponible dans les paramètres avancés d'application.




Cette option est cochée par défaut.

Lorsqu'elle est cochée, on ajoute toujours la valeur de 0.000001, ce qui rend tout arrondi à plus de 5 décimales faux.

Si on la décoche,

cette valeur de 0.000001 n'est plus ajoutée, les résultats sont justes.

Exemple :

Après le calcul, la formule donne 12,3456

Si la case cochée est :

  • Aucune (quel que soit le niveau) : le résultat rendu est 12,3456

  • Au plus près :
    • niveau : 0,1 le résultat rendu est 12,3
    • niveau : 0,01 le résultat rendu est 12.35

  • Inférieure :
    • niveau : 0,1 le résultat rendu est 12,3
    • niveau : 0,01 le résultat rendu est 12,34
    • niveau : 1 le résultat rendu est 12

  • Supérieure :
    • niveau : 0,1 le résultat rendu est 12,4
    • niveau : 0,01 le résultat rendu est 12,35
ARRONDI(N,A,B) :

Permet d'obtenir des arrondis dans des calculs pour des valeurs qui ne sont pas le résultat de la formule

Paramètres :

N : montant à arrondir

A : type d'arrondi : "" (au dessus)

B : niveau d'arrondi : "0,01", "1", etc...

ABS(X)

Retourne la valeur absolue de "X".

TRUNC(X)

Retourne la valeur tronquée à l'unité de "X".

EXP(X)

Retourne la valeur exponentielle de "X".

PUISS(X, Y)

Retourne la puissance de "X" à la puissance "Y"

LOGN(X, Y)

Retourne le logarithme base "X" de "Y".

LN(X)

Retourne le logarithme népérien de "X".

RACINE(X, Y)

Retourne la racine X de Y.
