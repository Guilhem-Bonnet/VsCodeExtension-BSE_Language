Variables locales

Contrairement aux variables temporaires, les variables locales sont spécifiques au script ou à la formule de calcul dans laquelle elles ont été créées (non globales).

LOCAL format nomvar incomplet

Déclaration de la variable locale.

Paramètres :

LOCAL : commande permettant d'indiquer que l'on va déclarer une variable locale

format : il est nécessaire préalablement à leur utilisation pour stocker une valeur de les déclarer en définissant leur format.

  • LOCALA (ou LOCAL$) permet de déclarer une variable au format alphabétique..
  • LOCALN (ou LOCAL) permet de déclarer une variable au format numérique
  • LOCALX permet de déclarer une variable au format booléen
  • LOCALD permet de déclarer une variable au format date

nomvar : nom donné par l'utilisateur à la variable (ne doit contenir que des lettres et des chiffres, sans caractères spéciaux, ni espaces)

!nomvar incomplet

Utilisation de la valeur d'une variable locale.

Le nom donné à la variable doit être précédé d'un '!' (point d'exclamation)
Son utilisation est la même que pour une variable locale, si ce n'est qu'il n'est pas nécessaire de redéfinir le format de la variable lors de l'attribution d'une valeur.

Exemple :

LOCALA TEST // déclaration variable TEST au format alphabétique
!TEST:= "variable" // attribution valeur
IA$:=!TEST + " locale" // utilisation de la variable dans une formule
=IA

Les variables locales à un calcul (à un script) ne sont visibles qu'au sein de cette formule

Il est possible également de créer des variables globales à une session de calcul

Une session de calcul se définit comme un environnement de calcul homogène et indépendant dans lequel sont exécutées les formules.

Il y a une session de calcul par défaut à l'application mais par exemple, chaque table a sa propre session de calcul

Idem pour les scripts automate, chaque script a sa propre session

Les variables IA -> IZ sont propres à une session et sont "globales" puisqu'elles sont exploitables dans les formules de calcul intégrées dans une formule ou un script.

Les variables locales, déclarées par l'instruction "local" dans une formule ou un script, ne sont pas visibles des autres formules et sont détruites à la fin du script (ou de la formule de calcul)

On peut donc créer des variables globales à une session, qui sont conservées en mémoire, qui peuvent ainsi être initialisées dans une formule ou un script et exploitées dans une autre formule

Syntaxe :

global variable (ou global$, globalD, globalX, globalN)
Ensuite, même syntaxe que pour les variables locales : !variable

Remarque : Si on crée une variable globale dans un calcul ou un script qui appelle une formule ou la même variable est déclarée locale (même nom), ce sera la variable locale qui sera manipulée dans le calcul et non pas la variable globale de la session.
