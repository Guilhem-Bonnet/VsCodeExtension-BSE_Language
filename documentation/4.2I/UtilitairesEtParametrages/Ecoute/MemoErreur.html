Envoi de mémo en cas d'erreur (uniquement sur les bases en Interbase ou FireBird)

Il est possible de demander à Ecoute.EXE d'envoyer un mémo automatiquement à utilisateur donné, sur une base donnée, en cas d'erreur.
Pour cela, il faut paramétrer une section [memo] dans le fichier config.ini utilisé par Ecoute.EXE

Section [memo] du fichier INI

Voici un exemple de section [memo], suivi d'un descriptif de chaque clé :
[Memo]
Destinataire=SUPERVISEUR
Base=C:\BELAIR3\APPDEV\APPDEV.IDB
User=BELAIR
Password=belair
  • Destinataire : cette clé permet de préciser le destinataire du mémo. Il faut de préférence choisir un destinataire qui est régulièrement connecté, sous peine de voir s'accumuler les messages si par exemple c'est la composante de traitement de fichier qui est en erreur, avec un automate d'import déjà utilisé ailleurs
  • Base : indique le chemin de la base où on doit ajouter le mémo. La base doit être une base Interbase, et le chemin doit être au format interbase : serveur:chemin.
  • User : code utilisateur pour se connecter à la base BELAIR. Généralement, le user BELAIR est un utilisateur déclaré et ayant le droit d'ajouter des mémos à la base.
  • Password : mot de passe pour l'utilisateur précisé.

On peut voir dans l'exemple que chaque erreur survenue déclenchera l'envoi d'un mémo à l'utilisateur SUPERVISEUR, dans la base interbase APPDEV.IDB de l'ordinateur qui exécute Ecoute.EXE (pas de serveur précisé).
